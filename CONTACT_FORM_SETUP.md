# お問い合わせフォーム 設定マニュアル

## 概要

`contact.html` は **Formspree**（外部のフォーム受信サービス）を使って送信する実装が済んでいます。
サーバー側の設定は不要です。

**✅ Formspree登録・フォーム作成・ID設定はすべて完了済みです。**
エンドポイント：`https://formspree.io/f/mojkaodq`
受信先：`info@water-fall.jp`

---

## デプロイ手順

### ステップ1：サーバーにアップロードして動作確認（これだけでOK）

1. フォルダごとサーバーにアップロード
2. ブラウザでサイトを開き、テスト送信を実施
3. Formspreeのダッシュボードと、設定したメールアドレスの両方に届くことを確認

---

## 送信されるデータ項目

| フィールド名（name属性） | 内容 |
|---|---|
| `name` | お名前（必須） |
| `company` | 会社名・団体名 |
| `email` | メールアドレス（必須） |
| `phone` | 電話番号 |
| `inquiry_type` | お問い合わせの種類（必須） |
| `event_scale` | 想定イベント規模 |
| `event_date` | 予定時期 |
| `budget` | ご予算の目安 |
| `message` | お問い合わせ内容（必須） |
| `referral` | このサイトを知ったきっかけ |

---

## 補足：フォームの動作について

- 送信ボタンを押すと「送信中…」に変わりロックされ、2重送信を防止します
- 送信成功後：フォームが非表示になり、同じページ内に完了メッセージを表示します（ページ遷移なし）
- 送信失敗時：アラートを表示してボタンを再度有効化します

## よくある質問

**Q. ドメインやサーバーの種類は関係ありますか？**
A. ありません。Formspreeはどんなサーバー・ドメインでも動作します。

**Q. スパムが増えた場合は？**
A. Formspreeダッシュボードの「Settings」からreCAPTCHAを有効化できます。

**Q. 受信メールアドレスを変更したい場合は？**
A. Formspreeダッシュボードの「Email Notifications」から変更できます。contact.htmlの修正は不要です。
